## Hands-On Title

SDLC vs TDLC — V-Model & Agile QA Integration

# Objective

The objective of this hands-on is to understand the relationship between Software Development Life Cycle (SDLC) and Testing Development Life Cycle (TDLC).

This exercise focuses on V-Model mapping, testing phases, entry and exit criteria, Agile QA practices, and Shift-Left testing principles.

---

# Task 1 - V-Model Mapping

## Objective

Understand how development phases are connected with corresponding testing phases using the V-Model approach.

---

## Features Implemented

- Created a complete V-Model diagram showing:

  - Requirements
  - System Design
  - Architecture Design
  - Module Design
  - Coding

  mapped with:

  - Acceptance Testing
  - System Testing
  - Integration Testing
  - Unit Testing

- Documented SDLC and TDLC phase mapping.

- Explained test artifacts produced during each development phase:

  - Acceptance Test Plan
  - System Test Plan
  - Integration Test Cases
  - Unit Test Cases

- Defined entry and exit criteria for:

  - Unit Testing
  - Integration Testing
  - System Testing
  - Acceptance Testing

- Identified early QA engagement points:

  - Requirements Review
  - Architecture Design Review

---

# Task 2 - Agile QA and Shift-Left Testing

## Objective

Understand QA involvement in Agile development and the importance of testing earlier in the SDLC.

---

## Features Implemented

- Explained problems with traditional Waterfall testing:

  - Late defect detection
  - Requirement misunderstanding
  - Limited testing time

- Documented QA responsibilities in Agile ceremonies:

  - Sprint Planning
  - Daily Standup
  - Sprint Review
  - Sprint Retrospective

- Explained Shift-Left testing practices:

  - Requirement review for testability
  - Writing test cases before development
  - Static code analysis
  - API contract testing

- Created acceptance criteria using Gherkin format:

  - Successful course creation
  - Duplicate course code validation
  - Missing required fields validation
